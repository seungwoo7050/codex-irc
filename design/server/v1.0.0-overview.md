# design/server/v1.0.0-overview.md

## 목적
- v1.0.0은 신규 기능을 추가하지 않고 기존 범위(PASS/NICK/USER~MODE/REHASH)를 안정화한다.
- 계약(`design/protocol/contract.md`)·구현·테스트 간 불일치를 제거하고, 누구나 CLONE_GUIDE만 따라도 동일한 동작을 재현하도록 문서를 정리한다.

## 정합성 정리
- 계약을 v1.0.0 기준으로 동결하고 지원/미지원 명령을 명확히 분리했다.
- PASS 오류 시 연결 종료, 송신 큐 초과 시 즉시 종료, 레이트리밋 드롭 후 `439` 반환 등 구현 동작을 문서에 반영했다.
- 채널 이름 규칙/모드(+i/+k/+t/+o/+l) 정책을 계약·테스트 모두에 명시해 혼선을 제거했다.

## 테스트 강화
- 핵심 흐름을 한 번에 검증하는 E2E 스모크를 추가해 아래를 확인한다.
  - 등록(PASS/NICK/USER)과 환영 numeric
  - 채널 JOIN/PART 브로드캐스트
  - 채널 PRIVMSG 브로드캐스트
  - 모드(+k 또는 +i) 정책에 따른 JOIN 거부/허용
- 기존 E2E(test_modes 등)와 함께 실행되어 계약 불일치를 조기에 탐지한다.

## 사용 가이드 보강
- CLONE_GUIDE에 설정 예시, REHASH 절차, 핵심 스모크 절차를 단계별로 정리해 새 기여자가 그대로 재현할 수 있게 했다.

## 남은 한계
- WHO/WHOIS/IRCv3 확장, TLS/서버 링크 등은 지원하지 않는다(계약·README·VERSIONING에 명시).
